# market-district

The **Market** area's backing service — a Go order-book / matching engine exposed
over gRPC that emits trade-settlement events on Kafka. It is qtown v2's
**distributed-systems proof**: concurrent matching, a Protobuf gRPC contract, and
an event backbone with at-least-once delivery + idempotent consumers.

## What it does

- Maintains a per-resource limit order book (in-memory) and matches crossing
  orders (`internal/orderbook`).
- Serves `MarketDistrict` gRPC (`PlaceOrder`, `GetOrderBook`, `PriceFeed`, …) on
  **:50051** (`internal/grpc`, registered in `cmd/server/main.go`).
- On each match, emits **one `qtown.economy.trade.settled` event per
  counterparty** to Kafka (`internal/kafka`).

## Contract

- **gRPC:** defined in `proto/qtown/market.proto`, code-generated by `buf` into
  the committed `gen/go` module and consumed via `replace qtown/proto =>
  ../../gen/go`. Regenerate with `make proto` (a CI drift gate enforces that the
  committed `gen/` matches).
- **Kafka — produces** `qtown.economy.trade.settled`, single-sided per
  counterparty:

  ```json
  {"npc_id": 1, "gold_delta": -50, "resource": "wood",
   "price": 10, "quantity": 5, "trade_id": "trade-1"}
  ```

  A match emits two messages: the buyer (`gold_delta < 0`) and the seller
  (`gold_delta > 0`), sharing one `trade_id`. Consumers (town-core, tavern,
  library) key idempotency on `(trade_id, npc_id)`.

## Run

```bash
# build / unit tests / vet
cd services/market-district
go build ./...
go test ./... -race

# run the server (needs Kafka for settlement emission; best-effort otherwise)
KAFKA_BROKERS=localhost:9092 go run ./cmd/server

# end-to-end (real gRPC + real Kafka) — start Kafka first, then:
docker compose -f ../../docker-compose.deps.yml up -d kafka
KAFKA_BROKERS=localhost:9092 go test -tags e2e -run TestE2E ./internal/grpc/
```
